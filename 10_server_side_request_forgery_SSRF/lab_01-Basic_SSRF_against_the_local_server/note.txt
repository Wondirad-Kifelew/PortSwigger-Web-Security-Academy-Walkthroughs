Basic SSRF against the local server

this web app has a SSRF vulnerability in stock check feature. while clicking the stock check, it sends a get req to some endpoint. so i changed the endpoint with GET http://localhost/admin and it worked but deleting carlos from that page didnt - It says that only admins or local hosts can delete it. This means that i should do the fetching the user and deleting carlos in one step. So i did using  GET http://localhost/admin/delete?username=carlos and done
